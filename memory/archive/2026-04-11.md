## 8:04 AM (BRT) — Critical Fix: Scraper Hangs on body.innerText()

### Problem
Scraper hangs indefinitely. The `page.locator('body').innerText()` on line ~231 of server.js waits for Airbnb's JS to finish DOM mutations — which never completes on their heavy React page.

### Fix Applied
Replaced the hanging `innerText()` call with a timeout-protected version:
```javascript
const bodyText = await page.evaluate(() => {
  return new Promise((resolve) => {
    const timeout = setTimeout(() => resolve(''), 5000);
    document.fonts.ready.then(() => {
      clearTimeout(timeout);
      resolve(document.body.innerText);
    });
  });
});
```
This gives it 5 seconds max, then returns empty string if not ready. All downstream code (price regex, description, details) falls back gracefully.

## 03:00 UTC — Scraper Anti-Detection Status

### Root Cause Confirmed
Airbnb detects headless Chrome via JavaScript. After `domcontentloaded`, their JS checks `navigator.webdriver` (returns `true` for headless) and redirects to 404 "Oops" page. The real listing content exists in the initial HTML but gets wiped by client-side JS.

### Evidence
- `curl` with Windows Chrome UA + sec-fetch headers → returns full listing HTML with price/bedrooms data (314KB)
- Playwright headless Chrome → gets redirected to 404 "Oops" page (same URL, status 200)
- `navigator.webdriver` is `true` in headless mode — Airbnb detects this
- `addInitScript` override of `navigator.webdriver` doesn't work — Airbnb has additional fingerprint checks

### Failed Solutions
1. playwright-extra + stealth-plugin — not installed (package issue)
2. undetected-chromedriver — fails on arm64 macOS ("Bad CPU type in executable")
3. Manual addInitScript to override webdriver/plugins/languages — Airbnb detects anyway

### What Works
- `curl` returns real listing HTML (314KB) with listing data
- Alternative: use Puppeteer instead of Playwright (might have better stealth support)

### Alternative Approaches to Try

2. Use `playwright` with `--headless=new` flag (more stealthy)
3. Use a proxy + real browser headers
