### How to Start All Servers

Run these commands in order:

```bash

cd ~/.openclaw/workspace/projects/vacation-rental-deploy && npx serve -l 9000 .

# 2. User site (port 9100)
cd ~/.openclaw/workspace/projects/vacation-rental-deploy/sites/avidmollison1newsiteadmin2gmailcom && npx serve -l 9100 .

# 3. Airbnb Scrape API (port 6900)
cd ~/.openclaw/workspace/projects/vacation-rental-deploy/sites/davidmollison1newsiteadmingmailcom/template/Airbnb\ Import\ Template && python3 api_server.py
```

### Airbnb Import Feature

**What it does:**
- User pastes Airbnb listing URL in the Setup modal
- Clicks "Import from Airbnb" or "Test Import"
- Scrapes real data from Airbnb using Selenium
- Saves to Supabase (user_sites + property_images tables)
- Updates the hero image on the page

**How to test:**
1. Go to http://localhost:9100
2. Setup modal appears automatically
3. Click "Airbnb profile import" radio button
4. Enter URL or click "Test Import"
5. Watch the spinner → see success preview

### Supabase Tables Used

- `user_sites` - stores property_source_url, property_title, property_hero_image, property_gallery, etc.
- `property_images` - stores individual images with is_main, is_featured, position

### Known Issues / Notes

1. **CORS**: The API server uses `flask-cors` to allow cross-origin requests from localhost:9100
2. **Selenium**: Requires Chrome/Chromedriver installed. Currently runs headless.
3. **Production**: Need to deploy API to Vercel Railway for production use (CORS won't work from production URL to localhost)
4. **Memory**: All file paths use ~ (home directory)
